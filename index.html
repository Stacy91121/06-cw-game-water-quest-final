ScalperBlock — Transaction Verification & Risk Scoring

Protect inventory and ensure fair access by verifying user purchase intent with a high-performance Risk Score API and contextual behavioral analysis.

View API Features

Merchant Dashboard

  • Live requests: 128
  • Avg latency: 34ms
  • Alerts: 2

Core Capabilities

Real-Time Scoring

/score/transaction — sub-100ms JSON response containing risk_score, recommended_action, and breakdown.

Multi-Factor Engine

IP reputation, velocity, fingerprinting, and behavioral anomaly detection are combined with weighted aggregation.

Management & Monitoring

Rule configuration UI, feedback capture for false positives/negatives, and continuous monitoring/alerting.

API Example — /score/transaction

Post a JSON payload. The API requires authentication (API Key or JWT). Example payload and a sample simulated response are shown below.

Request (JSON)

{
  "ip_address": "198.51.100.12",
  "user_agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64)...",
  "timestamp": 1699999999000,
  "product_id": "SKU-1234",
  "session_id": "sess-abc-001",
  "checkout_speed_ms": 1500
}

Sample Response

{
  "risk_score": 42,
  "recommended_action": "ALLOW",
  "breakdown": {
    "ip_score": 10,
    "velocity_score": 5,
    "fingerprint_score": 12,
    "behavior_score": 15
  }
}

Latency target: <100ms. Auth required for production.

Architecture & Diagrams

Backend: Java + Spring Boot (ApiEndpoint, RiskScoringEngine, RuleService, FeedbackService, LoggingService). Database: PostgreSQL with Users, Tasks, Items, Chats, Messages tables.
